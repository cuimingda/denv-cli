# ADR-004: 外部依赖封装与注入策略

## 决策标题
外部依赖统一在 `internal/infra` 与 `runtime` 接口中管理。

## 背景
`homebrew` 等外部命令在不同机器差异大，若散落在业务层会导致结果不可复现、难以测试。

## 可选方案
1. 在 `cmd` 或 `domain` 直接执行 `exec.Command`，通过条件判断兼容差异。
2. 将依赖判断、执行与测试替身集中到 `infra/runtime` 与路径策略模块。
3. 使用第三方库一次性抽象所有系统命令。

## 选择方案
选择方案 2。

## 选择理由
该策略匹配现有运行时接口与测试替身能力，便于跨环境注入不同执行器，同时保持业务规则无副作用。

## 未来可能调整条件
若要支持远程执行或容器内执行，可在 `infra` 下新增 Runtime 实现并通过既有注入点接入。
