# ADR-006: 以 gofmt/go vet/go test 构建合并护栏

## 决策标题
将 `gofmt`、`go vet`、`go test ./...` 设为 CI 进入主分支的最低标准。

## 背景
没有统一自动化护栏时，未来接手者可能只看到“能编译”而忽视格式、静态检查和测试契约，导致小改动触发结构退化或行为回归。

## 可选方案
1. 仅在本地要求测试，CI 不强制。  
2. CI 仅运行 `go test`。  
3. CI 强制 `gofmt`、`go vet`、`go test ./...`，并提供 `golangci-lint` 作为可选告警。

## 选择方案
选择方案 3。

## 选择理由
该方案把可传承的最低质量门槛固定到执行层；`go test` 提供功能回归约束，`gofmt/vet` 降低代码可读性和静态风险，`golangci-lint` 作为可选项可逐步收紧。

## 未来可能调整条件
当仓库规模扩大或引入更多静态规则时，可开启 `golangci-lint` 为强制项并新增自定义 lint 规则。
