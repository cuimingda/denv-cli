# ADR-003: 统一错误与退出码策略

## 决策标题
统一错误与退出码策略，收敛到“参数失败即停、业务失败可复现、CLI 统一非零退出码”。

## 背景
项目被脚本化调用，输出和退出码不稳定会直接影响自动化判断；需固定失败语义以减少接手者误判。

## 可选方案
1. 所有错误直接 panic，交由 runtime 处理。
2. 在每个命令点自行打印并退出，策略分散。
3. 统一在入口层定义失败路径映射，并确保关键失败场景有测试锚点。

## 选择方案
选择方案 3。

## 选择理由
方案 3 使非零退出码只由有限入口定义，便于接手者依赖测试行为确认稳定性。新增失败场景时先写契约测试，后改路径，降低回归风险。

## 未来可能调整条件
当需要更细粒度错误码体系（如分场景返回不同退出码）时，可扩展统一映射层，而非分散到命令实现。
