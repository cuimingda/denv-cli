# ADR-002: 统一命令与输出边界

## 决策标题
CLI 命令层只承担参数、帮助、输出与退出码，业务行为由服务层组织。

## 背景
`cmd` 层包含大量契约测试，若业务逻辑直接落在命令实现中，用户行为很容易在参数解析或文本输出上漂移，接手者难以验证改动后果。

## 可选方案
1. 将更多业务逻辑下沉到 `cmd`，以缩短服务层代码。
2. `cmd` 仅转发 flag 到服务层，错误和输出也由服务层统一控制。
3. 维持三层边界：`cmd` 负责协议层，`internal/service` 负责编排。

## 选择方案
选择方案 3。

## 选择理由
该方案与现有测试层次最匹配：`cmd` 测试聚焦输入/输出契约，`internal` 测试聚焦核心数据流，降低未来多人协作时的回归风险。

## 未来可能调整条件
若新增 Web/API 入站，不新建新规则层次，而在外部新增入口层复用同一 `service/domain/infra` 契约。
