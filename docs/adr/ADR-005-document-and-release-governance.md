# ADR-005: 文档与发布流程作为可传承主干

## 决策标题
将项目可传承能力依赖于 `README + CONTRIBUTING + docs/* + CHANGELOG` 的联合机制。

## 背景
历史信息不稳定，新执行者只能从仓库持续恢复上下文。单点文档无法覆盖架构约束与发布节奏，容易在交接后出现边界偏移。

## 可选方案
1. 仅补充 README 与单份开发文档。  
2. 将所有说明放在 `docs/project-overview.md`。  
3. 使用分层治理：`README` 负责上手，`CONTRIBUTING` 负责规则，`project-overview` 负责结构图谱，`CHANGELOG` 记录每轮变更。

## 选择方案
选择方案 3。

## 选择理由
分层文档使新执行者能在不看源码上下文的情况下快速执行决策：先快速理解边界，再查验约束，再按变更类型更新记录。该结构覆盖“为何改、改在哪、改了什么”三类可传承问题。

## 未来可能调整条件
若项目规模进入多人长期并行开发，新增 `docs/adr` 索引页与发布模板表单以减少人工同步成本。
